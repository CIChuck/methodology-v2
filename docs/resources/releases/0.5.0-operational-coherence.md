## Summary

0.5.0 turns the operational-coherence decision record into working methodology behavior. It closes
the gaps around phase-loop ownership, runtime gate transitions, strict schema-2 lifecycle events,
distribution/backfill safety, hook preservation, deterministic metrics, active documentation, and
current examples.

## As-built scope

Delivered:

- lifecycle registry and generated shell contract for G0-G9, G5 checkpoints, event profiles,
  version targets, approval ownership, and release validation rules;
- strict checker controls for project identity, C1/C2/C3 criteria, G3 verification specification,
  tactical task IDs, phase evidence, gate-log event evidence, late-gate artifacts, G8 deployment
  approval, and G9 terminal close-out;
- runtime transition commands for major gate closure, phase checkpoints, phase exit, G8 deployment
  approval, and terminal close-out;
- transaction-style write helpers with backups, rollback, dry-run, and noninteractive answers-file
  support;
- manifest-driven installer and safe backfill command with byte-preserving authority import;
- pre-commit hook installation that preserves existing hooks and supports uninstall;
- deterministic metrics for timing, traceability row shape, and value-review disposition without
  claiming unimplemented escape-rate metrics;
- current C1 executable example and current C2 planning example, with legacy examples preserved
  under a versioned historical directory;
- Ubuntu/macOS GitHub Actions matrix definition for methodology validation jobs.
