# GenDev `0.1.0-baseline` Release Notes

Status: Release-prep candidate
Release tag: `v0.1.0-baseline` after release-prep merge
Prepared: 2026-06-10

## Purpose

`0.1.0-baseline` is the hardened pre-1.0 GenDev methodology baseline. It is intended to be cloned,
initialized, and used as the starting point for AI-assisted product development with a human team
member and one or more coding agents.

This release is not an implemented product. It is a methodology baseline.

## Release Scope

This baseline includes:

- reusable methodology authority under `docs/methodology/`;
- root agent instructions in `AGENTS.md`;
- project initialization through `scripts/init-project.sh`;
- active project skeleton under `docs/project-template/`;
- methodology validation through `scripts/check-methodology.sh`;
- diff-aware guard binding through `scripts/methodology-guard.sh`;
- optional local hook installation through `scripts/install-hooks.sh`;
- on-demand metrics through `scripts/methodology-metrics.sh`;
- GitHub Actions reference binding in `.github/workflows/methodology.yml`;
- practitioner guide under `docs/resources/practitioner-guide/`;
- non-authoritative examples under `docs/resources/examples/`;
- advisory assessment history under `docs/resources/releases/0.1.0-baseline-provenance/`.

## Known Limits

- This is a pre-1.0 baseline. It should be exercised in real downstream projects before declaring a
  stable `1.0.0` methodology release.
- The GitHub Actions workflow is a reference binding, not a platform requirement.
- Tool-specific guidance exists for Codex and Claude Code only. Other tool addenda can be added
  later without changing the core methodology.
- `docs/resources/releases/0.1.0-baseline-provenance/` remains advisory. Ratified methodology authority lives in `AGENTS.md`,
  `README.md`, `docs/methodology/`, `docs/project-template/`, and the practitioner guide.

## Tagging

After this release-prep patch is merged to `master`, create and push the release tag:

```bash
git checkout master
git pull --ff-only
git tag -a v0.1.0-baseline -m "GenDev hardened pre-1.0 baseline"
git push origin v0.1.0-baseline
```

Use `v0.1.0-baseline` as the clone baseline for the first real downstream project trials.
