## Summary

GenDev 1.0.1 is the first published release of the GenDev 1.0 production line.
The 1.0.0 candidate completed the production release work packages and the
documentation consistency remediation but was never tagged or published. An
independent review of the candidate produced findings that were remediated on
the 1.0.1 release branch. 1.0.1 therefore carries the full 1.0 feature set:
truthful release identity, agent-operable workflows (`gendev-doctor.sh`,
`project-state.sh`, `new-artifact.sh`), coherent project scaffolding, full
lifecycle validation, and production adoption guidance.

1.0.1 adds one substantive change beyond remediation: a declared and enforced
prerequisite contract. Ripgrep joins bash, git, and python3 as a required tool.
Prerequisites are verified by the doctor, enforced at project initialization,
and guarded at point of use, so a missing tool fails loudly instead of passing
silently.

## Changes Since The 1.0.0 Candidate

- Release identity promoted to 1.0.1 across all active surfaces, with the
  never-published 1.0.0 candidate recorded as superseded.
- Prerequisite contract declared (bash, git, python3, ripgrep), verified by
  `gendev-doctor.sh`, enforced by `init-project.sh`, and guarded in
  rg-dependent scripts. The grep emulation shim was removed from the test
  helpers; ripgrep is now required, not emulated.
- `check-doc-coherence.sh` fails loudly when ripgrep is missing. Its negative
  assertions can no longer pass silently on a machine without ripgrep.
- Remote CI installs ripgrep explicitly on all runner platforms.
- Release-coherence case RC-008 is scoped explicitly to the historical 0.5
  review, and a new case asserts the 1.0.1 independent review artifact records
  a non-pending result.
- Documentation coherence review-date assertions are structural (a dated
  `Last reviewed:` line must exist) rather than pinned to a single date.
- Independent review of the 1.0 candidate is recorded in
  `docs/resources/evolution/1.0.1-independent-review.md`.

## Prerequisites

```text
bash 4 or later
git
python3
ripgrep (rg)
```

Verify with:

```bash
./scripts/gendev-doctor.sh
```

## Publication Gates

1. Local validation passes.
2. Remote CI is configured and observed green on the release branch.
3. Branch protection is configured before publication.
4. Independent review records a non-pending result.
5. Release notes, release index, lifecycle registry, generated contract, README,
   adoption guide, and practitioner guide agree on release identity.
6. Annotated tag `v1.0.1` is created at the reviewed release commit.
7. GitHub Release `GenDev 1.0.1 Production Release` is published.

Publication procedure: the final publication commit flips the release index and
lifecycle registry to published state, the annotated tag is created on that
commit immediately, and the commit and tag are pushed together so remote
validation observes them atomically.
