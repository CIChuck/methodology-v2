# 0.5.0 Operational Coherence Migration Guide

Status: Release-prep candidate
Target version: 0.5.0-operational-coherence

## Migration principle

Migration to 0.5 is opt-in. Existing projects may remain pinned to 0.1 through 0.4. A project does
not regress solely because 0.5 exists, and historical events are append-only. Missing facts remain
unknown; they are never inferred to make a report look complete.

## Assessment output

A 0.5 migration assessment must report:

```text
current methodology version and detected schema
target version
current major gate and phase position
missing canonical directories/artifacts
floating or stale canonical paths
missing/mismatched project identity
incomplete historical approval events
unbound phase evidence classes
missing tactical task IDs and legacy mappings
coverage-policy gap
NAA directives pinned to old definition
active amendments/stale artifacts
recommended adoption point
required human approvals
checker/coherence results and real exit status
```

## Non-destructive adoption flow

1. Run assessment only; do not modify project files.
2. Review missing directories, identity fields, fixed paths, lifecycle artifacts, phase evidence,
   task IDs, schema status, and version claims.
3. If the human approves, create missing directories/scaffolds non-destructively.
4. Preserve imported authority bytes. Do not overwrite accepted customer/project documents.
5. Add reconciliation events that map historical evidence to 0.5 evidence classes. Do not edit
   historical event text.
6. Decide active mid-phase posture: finish under the pinned old directive or amend/reissue under
   0.5.
7. Validate checker/coherence results.
8. Change `project.methodology_version` last, after validation and approval.

## Legacy rules

- New events after migration begins use schema version 2.
- Deprecated manifest fields may be read for one release, but new templates omit them.
- Reconstructed artifacts must be labeled reconstructed and include provenance and limitations.
- Legacy warning mode is version-bound compatibility, not a permanent bypass.
- Missing approvals, revisions, task IDs, or evidence remain unresolved until a human records the
  disposition.
