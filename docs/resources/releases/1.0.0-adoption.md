# GenDev 1.0 Adoption Guide

Status: Draft for 1.0 release validation
Target version: 1.0.0
Release notes: `docs/resources/releases/1.0.0.md`

## Purpose

This guide explains how to adopt GenDev 1.0 for a new project or for a repository
that already contains product artifacts. There are no existing GenDev projects
to convert as part of the 1.0 release; the adoption path therefore focuses on
clean initialization, explicit import of existing product authority, and
validation before implementation work proceeds.

## New Project Adoption

1. Clone or copy the GenDev 1.0 baseline.
2. Run project initialization:

   ```bash
   ./scripts/init-project.sh "Project Name"
   ```

3. Review `docs/project/project.yaml` and assign named human owners.
4. Run the methodology checker:

   ```bash
   ./scripts/check-methodology.sh
   ```

5. Start at G1 and advance only through approved gate transitions.

## Repository With Existing Product Artifacts

Existing vision, PRD, architecture, or governance material may be imported as
source material, but it is not automatically authoritative. The adopting team
must:

- identify the existing artifacts and their owners;
- preserve source material byte-for-byte where possible;
- decide which material is accepted, stale, superseded, or rejected;
- map accepted material into GenDev canonical artifacts;
- record human approval before using imported material as gate evidence;
- run validation before implementation work proceeds.

## Agent Operating Rule

An AI agent must not infer missing approvals or silently normalize existing
documents into authority. If source material is incomplete, contradictory, or
unowned, the agent records the gap and asks the human for disposition.

## First Validation

After initialization or import, run:

```bash
./scripts/check-methodology.sh
```

For release validation of the methodology repository itself, run the command set
listed in `docs/resources/releases/1.0.0.md`.
