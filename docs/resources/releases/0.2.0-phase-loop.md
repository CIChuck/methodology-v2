# Release 0.2.0 — Phase Loop

Status: Accepted
Date: 2026-06-12
Supersedes: 0.1.0-baseline

## Summary

Formalizes the interior of the G5 to G6 span as the GenDev phase loop: an
N-phase build driven by an accepted phase plan, with interior `G5.x`
checkpoints, a six-artifact phase document set, structured phase events, and a
promoted process guide. Gates remain G0 through G9; checkpoints are interior
addresses, not gates.

See docs/methodology/design/phase-loop-decision-record.md for the decisions and
rationale.

## Methodology version

This release is Status: Proposed. The project template stamp remains
`0.1.0-baseline` until the amendment is accepted; on acceptance the template
default advances to `0.2.0-phase-loop`. The stamp is intentionally not bumped
while the change is under review, so that an unmerged amendment does not assert
a version it has not yet earned.

After acceptance: methodology_version advances to 0.2.0-phase-loop for projects
adopting this release. Existing 0.1.0-baseline projects remain valid; phase-loop
manifest fields are optional.
