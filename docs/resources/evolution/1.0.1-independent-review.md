## Findings And Dispositions

### F-1: Publication gates open despite production framing

Severity: High. The release defined seven publication gates; branch
protection, independent review, tag creation, and GitHub Release publication
remained open while working language described the repository as a production
release. The repository's own documents stated this honestly.

Disposition: accepted as accurate state, not a defect in the documents.
Remediated by plan: the 1.0.1 release supersedes the never-published 1.0.0
candidate, this review closes the independent review gate subject to human
disposition, and the remaining gates are administrative actions recorded in
the 1.0.1 execution log.

### F-2: check-doc-coherence.sh failed open without ripgrep

Severity: High. The checker hard-depended on `rg` with no presence guard. On
a machine without ripgrep, every positive assertion reported a false error
and every negative assertion passed silently, because a missing engine was
indistinguishable from "no match found." The 0.5 portability work had removed
the ripgrep dependency from the test suites via a grep shim, but this new
operator script reintroduced the dependency unguarded.

Disposition: remediated on the 1.0.1 branch by policy plus enforcement.
Ripgrep is now a declared prerequisite (with bash 4+, git, and python3),
verified by the doctor, enforced at initialization, and guarded at point of
use. The grep shim was removed rather than extended: one regex engine
everywhere beats emulation drift. Regression case DOC-007 proves the
missing-rg failure mode stays loud.

### F-3: RC-008 name implied 1.0 review coverage it did not have

Severity: Medium. The release-coherence case named "independent review
artifact records a non-pending result" asserted against the historical 0.5
review document. A green suite could be read as verifying the 1.0 review
gate, which remained open.

Disposition: remediated. RC-008 is renamed to scope it to the 0.5 review, and
RC-010 asserts this 1.0.1 review artifact records a non-pending result.

### F-4: Date-pinned checker assertions

Severity: Low. The doc-coherence checker pinned `Last reviewed: 2026-07-11`
for the tool-specific practitioner notes, coupling every future review of
those files to a checker edit.

Disposition: remediated. The assertion is structural: a valid dated
`Last reviewed:` line must exist.

### F-5: Version identity ambiguity between 1.0.0 and 1.0.1

Severity: Medium. The remediation that produced the reviewed master state was
branded 1.0.1 in branch and log names while every active artifact declared
1.0.0, and no 1.0.1 release note existed.

Disposition: resolved by human decision. The publication version is 1.0.1;
the 1.0.0 candidate is recorded as superseded and never published. Release
identity was promoted across all active surfaces, and `check-methodology.sh`
now derives the current version from the lifecycle contract rather than
pinning a literal, removing this class of drift.
